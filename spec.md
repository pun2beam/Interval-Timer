# 繰り返しインターバルタイマー Webアプリ仕様書

## 1. 開発目的

ユーザーが指定した時間を1周期として、停止されるまで同じ時間のタイマーを繰り返し実行するWebアプリを作成する。

例として「2分」を設定した場合、次の動作を行う。

1. スタートボタンを押す。
2. 2分間の計時を開始する。
3. 終了10秒前から、10、9、8……1とカウントダウンする。
4. 2分経過時に終了音を鳴らす。
5. 間を空けず、直ちに次の2分タイマーを開始する。
6. ユーザーが停止するまで繰り返す。

単純に一定時間ごとに処理を実行するのではなく、ブラウザの処理遅延による時間のずれが累積しないようにする。

---

# 2. 想定利用環境

* PCおよびスマートフォンの一般的なWebブラウザ
* Google Chrome
* Microsoft Edge
* Safari
* Firefox
* インストール不要
* サーバー通信不要
* すべてブラウザ内で動作する静的Webアプリ

# 3. 技術要件

## 3.1 基本構成

特別な理由がなければ、以下の構成で実装する。

* HTML
* CSS
* TypeScriptまたはJavaScript
* 必要に応じてViteを使用
* 外部UIライブラリは原則として使用しない
* サーバー側処理は使用しない

Reactなどのフレームワークは必須ではない。小規模なアプリなので、保守性を損なわない範囲で最小構成とする。

## 3.2 時刻管理

タイマーの正確性を保つため、残り時間を1秒ずつ減算する方式にはしない。

各周期について、以下の絶対時刻を管理する。

* アプリ開始基準時刻
* 現在の周期番号
* 現在周期の開始予定時刻
* 現在周期の終了予定時刻

残り時間は次の考え方で毎回計算する。

```text
残り時間 ＝ 周期の終了予定時刻 － 現在時刻
```

画面更新には `requestAnimationFrame` または短い間隔の `setInterval` を使用してよいが、画面更新回数を時刻計測そのものには使用しない。

## 3.3 ずれの累積防止

次の周期の開始時刻を「前の周期が実際に終了処理を実行した時刻」から計算してはならない。

次の式を基本とする。

```text
第N周期の開始予定時刻
＝ タイマー全体の基準開始時刻
＋ 設定周期 × N
＋ シフト補正の累計
```

ブラウザの処理が一時停止または遅延しても、その遅延が次の周期へ累積しないようにする。

# 4. 画面構成

単一画面のWebアプリとする。

## 4.1 時間設定欄

周期の長さを以下の3個のプルダウンメニューで指定する。

* 時：0～23
* 分：0～59
* 秒：0～59

初期値は以下とする。

* 時：0
* 分：2
* 秒：0

最低設定時間は1秒とする。

「0時間0分0秒」ではスタートできない。

## 4.2 操作ボタン

以下のボタンを配置する。

### スタート

* 停止中に押すと繰り返しタイマーを開始する。
* 開始時点を第1周期の開始時刻とする。
* 実行中は無効化するか、「停止」ボタンと明確に区別する。

### 一時停止／再開

* 実行中に押すとタイマーを一時停止する。
* 一時停止中は残り時間が変化しない。
* 再開時は、一時停止していた時間分だけ、以降の全周期の予定時刻を後ろへずらす。
* 一時停止中はカウントダウン音を鳴らさない。

### 停止

* タイマーを完全に停止する。
* 周期番号、経過時間、シフト量などの実行状態をリセットする。
* 設定した時・分・秒は保持する。
* 停止後は再びスタートできる。

### 1秒＋シフト

* 現在の周期の終了予定時刻を1秒後ろへずらす。
* それ以降の全周期も1秒後ろへずらす。
* 周期そのものの設定時間は変更しない。
* 連打可能とする。
* 例：2分周期の途中で3回押した場合、今回および以降の周期境界を合計3秒後ろへずらす。

### 1秒－シフト

* 現在の周期の終了予定時刻を1秒前へずらす。
* それ以降の全周期も1秒前へずらす。
* 周期そのものの設定時間は変更しない。
* 連打可能とする。
* 操作によって現在周期の終了予定時刻が現在時刻以前になった場合は、直ちに周期終了処理を行う。

### リセット

* 停止と同じく実行状態を初期化する。
* 時間設定も初期値の「2分」に戻す。
* 誤操作防止のため、実行中に押された場合は確認ダイアログを表示してもよい。

## 4.3 表示項目

以下を大きく表示する。

### 残り時間

表示形式：

```text
HH:MM:SS
```

1時間未満でも、原則として次のように表示する。

```text
00:01:59
```

終了10秒前からは、秒数を特に大きく表示する。

例：

```text
10
9
8
...
1
```

### 状態

以下のいずれかを表示する。

* 停止中
* 実行中
* 一時停止中
* カウントダウン中

### 周期番号

現在実行している周期を表示する。

例：

```text
3周目
```

スタート直後を「1周目」とする。

### 設定周期

例：

```text
設定周期：2分00秒
```

### 累計シフト

例：

```text
シフト：+3秒
```

マイナスの場合：

```text
シフト：-2秒
```

### 次回終了予定時刻

ローカル時刻で表示する。

例：

```text
次回終了：14:32:10
```

# 5. タイマーの動作仕様

## 5.1 スタート時

1. 設定値を秒単位へ変換する。
2. 0秒でないことを確認する。
3. 現在時刻を基準開始時刻として保存する。
4. 周期番号を1にする。
5. 累計シフトを0秒にする。
6. 第1周期の終了予定時刻を計算する。
7. タイマー画面の更新を開始する。

## 5.2 通常動作

タイマー実行中は、少なくとも200ミリ秒以下の間隔で表示状態を確認する。

ただし、表示上の残り秒数は次のように算出する。

```text
表示残り秒数
＝ ceil((終了予定時刻 - 現在時刻) / 1000)
```

これにより、開始直後に2分設定が「1分59秒」と表示されることを防ぐ。

## 5.3 終了10秒前のカウントダウン

残り時間が10秒以下になったら、毎秒カウントダウンを行う。

* 10秒：音声または短い電子音
* 9秒：音声または短い電子音
* 以下同様
* 1秒：音声または短い電子音
* 0秒：周期終了音

同じ数字の音を複数回鳴らしてはならない。

ブラウザの描画更新が複数回行われても、各カウントダウン番号につき1回だけ再生する。

## 5.4 周期終了時

現在時刻が終了予定時刻以上になった場合、以下を実行する。

1. 周期終了音を鳴らす。
2. 周期番号を1増やす。
3. 次周期の開始予定時刻と終了予定時刻を絶対時刻から算出する。
4. カウントダウン済み状態をリセットする。
5. 間を空けず、次の周期の表示へ移る。

「終了音を鳴らし終わってから次周期を始める」のではなく、予定時刻上は直ちに次周期が始まる。

音声再生とタイマー進行は独立させる。

## 5.5 大幅な処理遅延からの復帰

PCのスリープ、スマートフォンの画面消灯、ブラウザのバックグラウンド制限などにより、複数周期分の時間が経過している可能性がある。

復帰時には、現在時刻から現在該当する周期番号を算出する。

例：

* 周期：2分
* 開始から7分経過
* 復帰時点では4周目の途中

この場合、終了した全周期について終了音を連続再生してはならない。

復帰時の動作は次のとおりとする。

* 過去の周期終了音は省略する。
* 現在時刻に対応する周期へ即座に移動する。
* 現在の周期番号と残り時間を正しく表示する。
* 復帰した時刻が周期境界の直後であれば、終了音を最大1回だけ鳴らしてよい。
* 経過した周期数を内部的に正しく反映する。

# 6. 音声仕様

## 6.1 初期実装

外部音声ファイルに依存せず、Web Audio APIを使用した電子音で実装してよい。

以下の2種類を区別する。

### カウントダウン音

* 短い音
* 1回
* 0.1～0.2秒程度
* 毎秒聞いても不快になりにくい音

### 周期終了音

* カウントダウン音より明確な音
* 2回または3回
* 次周期の開始を妨げない
* 非同期で再生する

## 6.2 音量

音量調整欄を設ける。

* 0～100％
* 初期値50％
* ミュート切替ボタンを設ける

## 6.3 音声利用制限への対応

ブラウザでは、ユーザー操作なしの音声再生が制限される。

スタートボタンを押した時点でAudioContextを開始または再開し、それ以降の音声が再生できるようにする。

音声再生に失敗してもタイマー自体を停止させてはならない。

---

# 7. 時間設定の変更

実行中は、時・分・秒のプルダウンを無効化する。

設定周期を変更する場合は、一度停止してから変更する。

実行中に周期設定を変更する機能は、初期バージョンには含めない。

---

# 8. シフト操作の詳細

## 8.1 シフトの意味

シフトは周期時間を変える操作ではなく、周期境界全体の位相を前後へ補正する操作とする。

例：

* 設定周期：120秒
* 元の終了予定：12:02:00
* 1秒＋シフトを3回押す
* 新しい終了予定：12:02:03
* 次周期終了：12:04:03

周期の長さは引き続き120秒である。

## 8.2 シフト表示

累計シフト量を符号付きで表示する。

```text
+0秒
+3秒
-2秒
```

60秒以上になっても、初期実装では秒単位表示でよい。

例：

```text
+75秒
```

## 8.3 キーボード操作

PC向けに以下のショートカットを実装する。

* Space：スタート、一時停止、再開
* Escape：停止
* → または `+`：1秒＋シフト
* ← または `-`：1秒－シフト

プルダウンや入力要素にフォーカスがある場合は、意図しないショートカット動作を避ける。

---

# 9. ブラウザバックグラウンド動作

ブラウザがバックグラウンドになると、JavaScriptのタイマー処理間隔が制限される場合がある。

そのため、次を守る。

* `setInterval` の呼び出し回数を時刻として信用しない。
* 常に現在時刻と終了予定時刻との差から残り時間を算出する。
* タブが再表示された際は `visibilitychange` イベントで即座に再計算する。
* `focus` イベントでも再計算する。
* PCのスリープ復帰後も時刻を再計算する。

Webブラウザの制約上、バックグラウンド中や端末スリープ中に音を正確な時刻に鳴らすことは保証できない。この制約はREADMEに記載する。

---

# 10. 設定保存

以下の設定を `localStorage` に保存する。

* 時
* 分
* 秒
* 音量
* ミュート状態

ページ再読み込み後も設定値を復元する。

実行中のタイマー状態については、初期バージョンでは保存しない。ページを再読み込みした場合は停止状態に戻す。

# 11. デザイン要件

* 操作方法が一目で分かること
* 残り時間を画面中央に大きく表示する
* スマートフォンでも操作しやすいボタンサイズにする
* 主要ボタンのタップ領域は最低44px四方とする
* 横幅320px程度の画面でも横スクロールを発生させない
* 明るい画面と暗い画面の両方で見やすくする
* OSのダークモード設定に自動対応する
* 色だけに依存せず、文字でも状態を示す
* カウントダウン中は視覚的にも明確にする
* 過剰なアニメーションは使用しない

6. スタート／一時停止／停止

# 12. アクセシビリティ

* すべてのボタンに明確なラベルを設定する
* プルダウンに `label` を関連付ける
* キーボードのみでも操作できる
* フォーカス位置を視覚的に表示する
* 状態変化を必要に応じて `aria-live` で通知する
* ただし、毎秒の残り時間をスクリーンリーダーへ通知して操作を妨げない
* 周期終了や一時停止など、重要な状態変化のみ通知する

# 13. エラーおよび境界条件

以下のケースを適切に処理する。

## 13.1 0秒設定

スタートを許可しない。

画面に以下のようなエラーを表示する。

```text
1秒以上の時間を設定してください。
```

## 13.2 1秒周期

1秒周期も動作させる。

ただし、終了10秒前カウントダウンは周期より長いため、次のルールとする。

* 周期が10秒以下の場合、開始時点からカウントダウン対象とする。
* 1秒周期では、開始時の「1」の音と終了音が過度に重ならないようにする。
* 1秒周期ではカウントダウン音を省略し、周期終了音だけを鳴らしてもよい。

## 13.3 シフトによる即時終了

マイナスシフトにより終了予定時刻が現在時刻以前になった場合、現在周期を終了させる。

複数周期分を一度に飛び越えた場合でも、終了音を大量に連続再生しない。

## 13.4 ボタン連打

* スタートボタンの連打で複数の更新ループを作らない。
* 停止を複数回押してもエラーにしない。
* シフトボタンを高速連打しても状態を失わない。
* 一時停止と再開を高速に切り替えても時刻計算を破綻させない。

# 14. 状態管理

最低限、以下の状態を管理する。

```ts
type TimerStatus = "stopped" | "running" | "paused";

interface TimerState {
  status: TimerStatus;
  intervalMs: number;
  baseStartTimeMs: number | null;
  currentCycle: number;
  currentCycleEndTimeMs: number | null;
  totalShiftMs: number;
  pauseStartedAtMs: number | null;
  totalPausedMs: number;
  lastPlayedCountdownNumber: number | null;
  volume: number;
  muted: boolean;
}
```

実装時により適切な構造へ変更してもよいが、次の情報は失わないこと。

* 絶対的な基準開始時刻
* 周期時間
* 現在周期
* シフト量
* 一時停止時間
* 再生済みカウントダウン番号

# 16. テスト要件

可能であれば、時刻計算部分をUIから分離し、自動テストを作成する。

最低限、以下を確認する。

## 16.1 基本動作

* 2分設定でスタートできる。
* 残り時間が正しく減少する。
* 終了時に次周期へ移る。
* 周期番号が増える。
* 停止するまで継続する。

## 16.2 カウントダウン

* 10から1まで各数字につき1回だけ音が鳴る。
* 描画更新が多くても同じ数字を重複再生しない。
* 周期終了時に終了音が鳴る。

## 16.3 ドリフト

疑似時計を使用して、次のケースをテストする。

* 更新処理が100ミリ秒遅れても次周期が100ミリ秒ずれない。
* 数秒処理が停止しても正しい残り時間に復帰する。
* 複数周期分停止しても現在周期を正しく算出する。

## 16.4 一時停止

* 一時停止中は残り時間が変化しない。
* 再開後は一時停止前の残り時間から続行する。
* 一時停止時間が後続周期にも反映される。

## 16.5 シフト

* ＋1秒で終了予定時刻が1秒後ろへずれる。
* －1秒で終了予定時刻が1秒前へずれる。
* シフト後も周期の長さは変化しない。
* 累計シフト量が正しく表示される。
* マイナスシフトで終了時刻を超えた場合、適切に次周期へ移る。

## 16.6 入力

* 0秒では開始できない。
* 1秒で開始できる。
* 最大値の23時間59分59秒で開始できる。
* 保存した設定が再読み込み後に復元される。

# 17. 完了条件

以下をすべて満たした時点で実装完了とする。

1. 時・分・秒をプルダウンで設定できる。
2. スタートすると同じ周期を停止まで繰り返す。
3. 終了10秒前からカウントダウンする。
4. 周期終了時に音を鳴らす。
5. 周期終了後、直ちに次周期へ移る。
6. 処理遅延が周期ごとに累積しない。
7. 一時停止、再開、停止が動作する。
8. 1秒単位で周期境界を前後へシフトできる。
9. PCとスマートフォンの両方で操作できる。
10. ページ内にJavaScriptエラーが発生しない。
11. READMEに起動方法とブラウザ上の制約を記載する。
